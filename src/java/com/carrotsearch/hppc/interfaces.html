Overview of HPPC's higher level interfaces and abstract classes.

Interfaces and container inheritance

HPPC's high-level interfaces are modeled to encapsulate the common functionality enforced from all the classes that implement a given interface. We tried to avoid imposing method names that would be confusing in the context of a given sub-class. For example, add() in the context of a Deque.

The resulting highest super-interface will not allow performing modifications to the collection, but will allow iterating through the collection, for example.
